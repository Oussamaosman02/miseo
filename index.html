Head

  • Meta Robots

    La tag de meta robots es predominante, igual que el robots txt, lo que significa que se seguirá al pie de la letra, a diferencia de otras tags como canonical que solo es un "a tener en cuenta"

    <meta name="robots" content="index,follow" />

    Por defecto es index,follow y los parametros pueden ser tanto index, follow como no index nofollow pero:¿Qué significa esto?

    • index: significa que se autoriza a que la página sea indexada en el motor de búsqueda
    • noindex: significa que no se quiere indexar la página en ningún motor de búsqueda.esta es útil para páginas que están sin acabar o ya sea una zona restringida.
    • follow: esto le indica al crawler o "araña" que puede seguir todos los links que encontremos en la página.
    • nofollow: esto le dices al crawler que no debe seguir los links que se encuentran en la página

    Puede haber combinaciones de:

    • noindex,nofollow: ni se indexa ni se siguen los links
    • index,nofollow: se indexa pero no se siguen los links
    • index,follow: que es la que viene por defecto, se indexa y si se permite seguir los links
    • all: esta es igual al index,follow solo que para vagos, así no escribes tanto
    • Todas las posibilidades-Google

  • Google tags
    • Google Bot

      <meta name="googlebot" content="noindex,nofollow" />

      Está es igual al robots, solo que específica de google, por si hubiera algún caso en el que quisieras tratar alguna página distinto en este motor de búsqueda

    • nositelinkssearchbox

      <meta name="google" content="nositelinkssearchbox" />

      Hay veces que cuando buscas en google un sitio, justo en la SERP aparece en eses sitio una barrita de búsqueda, para buscar en ese sitio, pues con esto lo evitas

    • notranslate

      <meta name="google" content="notranslate" />

      Esto evita que la página sea traducida

  • Canonical

    Una URL canónica es la url que (según el motor de búsqueda) es la más relevante sobre algunas duplicadas

    <link rel="canonical" href="https://example.com/products/phone" />

    Estas URL canónicas son muy útiles para que google pueda diferenciar las url originales de las duplicadas. Si hay URL duplicadas y no se indica cual es la canónica, google podría escoger mal o incluso no indexar ninguna de las dos

  • Meta Description

    La meta description sirve para describir el contenido de la página
    Esta descripción aparecerá debajo del título y la URL de tu página en la SERP

    <meta name="description" content="Entre 120 y 140 caracteres¡No olvides incluir tu palabra clave!">

    Tips
    • Escribelas como si fuera un anuncio
    • Lenguaje activo
    • Como aparecen en la SERP, deberían estar orientadas a obtener más clics
    • Alusiones a una de las 4 emociones más importantes: Contento, Triste, Enfadado/Disgustado o Asustado/Sorprendido
    • Tener descripciones únicas para cada página de tu web
    • Incluir tu palabra clave
    • Que coincidan con tu contenido
    • Solo una meta descripción
    • Estrategia USP: "lo que tienes que los competidores no tienen"

    En resumen, la meta description debería agregar una llamada a la acción clara, abordar un punto de dolor emocional u ofrecer un beneficio específico a los visitantes.

  • Title

    Un tittle es el título de la página que aparece directamente debajo de la URL en la SERP y en la pestaña del navegador
    Para establecer el título de una página, utiliza la etiqueta title en el HTML alrededor de tu texto

    <title>Este es el título </title>

    Tips
    • Indica la temática de la web
    • Haz que sea simple pero eficaz
    • Precisa y relevante sobre el contenido de la página
    • Incluye la palabra clave más relevante
    • Desencadenar un pensamiento o emoción
    • Palabras desencadenantes: adverbios interrogativos (cómo, qué, cuánto, dónde) o adjetivos (mejor, definitiva, etc)
    • 55-60 caracteres
    • Etiquetas únicas

    En resumen, las meta tittle deben tener la longitud adecuada, que ser únicas para cada página, describir concisamente el contenido de la página con la palabra clave objetivo y estar enfocadas a conseguir el clic

  • Open Graph

    El protoclo Open Graph fue desarrolado por Facebook y hace que tus páginas se vean mejor cuando son compartidas en redes sociales

    <meta property="og:title" content="Social Title for Cool Page" />

    Existen varios tipos, así como de twitter y otras redes, aquí nos centraremos solo en og

    • Titulo

      <meta property="og:title" content="Social Title for Cool Page" />
      En esta debes poner el título de tu página

    • Idioma

      <meta property="og:locale" content="es_ES" />
      En esta debes poner el idioma de tu página

    • Descripción

      <meta property="og:description" content="And a social description for our cool page" />
      Esta será una descripcion breve que se mostrará debajo de la foto y del título

    • Imagen

      <meta property="og:image" content="https://example.com/images/cool-page.jpg" />
      Está imagen debe ser representativa del contenido ya que se verá en grande

    • Url

      <meta property="og:url" content="https://exaple.com/example/" />
      En esta debes poner la url de tu página

    • Site Name

      <meta property="og:site_name" content="Example" />
      En esta debes poner el nombre de tu página, como el título

    • Imagen tamaño

      <meta property="og:image:width" content="500" /> <meta property="og:image:height" content="400" />
      En estas debes poner el tamaño del icono de tu página

  • Datos estructurados

    Estos facilitan a los crawlers a entender tu página.Se usa el vocubulario de Schema.org

    Por ejemplo, en una página de recetas, podrías marcar con datos estructurados los ingredientes, el tiempo y la temperatura de cocción, o las calorías, entre otros detalles.

    Implementarla en Next js 

    {
  "@context": "https://schema.org",
"@graph": [
  {
    "@type": "WebSite",
    "@id": "${URL}/#website",
    "url": "${URL}/",
    "name": "${title}",
    "description": "${short_description}",
    "potentialAction": [
      {
        "@type": "SearchAction",
        "target": {
          "@type": "EntryPoint",
          "urlTemplate": "${URL}/?s={search_term_string}"
        },
        "query-input": "required name=search_term_string"
      }
    ],
    "inLanguage": "es"
  },
  {
    "@type": "ImageObject",
    "@id": "$1{URL}/#primaryimage",
    "inLanguage": "es",
    "url": "$1{URL}/${primary_image}",
    "contentUrl": "$1{URL}/${primary_image}",
    "width": 900,
    "height": 432,
    "caption": "$1{image_figcaption}"
  },
  {
    "@type": "WebPage",
    "@id": "$1{URL}/#webpage",
    "url": "$1{URL}/",
    "name": "$1{meta_title}",
    "isPartOf": { "@id": "${URL}/#website" },
    "primaryImageOfPage": { "@id": "$1{URL}/#primaryimage" },
    "datePublished": "$1{}2018-02-20T10:32:57+00:00",
    "dateModified": "$1{}2021-07-27T11:25:43+00:00",
    "description": "$1{meta_description}",
    "breadcrumb": { "@id": "$1{URL}/#breadcrumb" },
    "inLanguage": "es",
    "potentialAction": [
      { "@type": "ReadAction", 
      "target": ["$1{URL}/"] }
    ]
  },
  {
    "@type": "BreadcrumbList",
    "@id": "$1{URL}/#breadcrumb",
    "itemListElement": [
      { "@type": "ListItem", 
      "position": 1, 
      "name": "Home" }
    ]
  }
  ]
}

    Cosas a tener en cuenta:

    • BreadcrumbList: Una BreadcrumbList es una ItemList que consta de una cadena de páginas web vinculadas, normalmente descritas usando al menos su URL y su nombre, y normalmente terminando en la página actual (sin URL).

      EL BreadcrumbList también podría ser así: 

      {
  "@type": "BreadcrumbList",
  "@id": "$1{URL}#breadcrumb",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "${URL}"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Blog",
      "item": "${URL2}blog/"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "$1{title}"
    }
  ]
},

    • Donde pone ${} es referente a la página general, está en todas, sin embargo el $1{} solo sirve para la página en cuestion
    • En el image Object, hay que poner el width y el height relativo a nuestra imágen
